2) Les variables

Notion de variable

Une variable doit être définie :

Les opérations élémentaires possibles sur les variables sont :

  • Déclaration
  • Affectation
  • Utilisation
  • Ré-affectation

Les types élémentaires possibles sont :

  • Les entiers (integer en PASCAL)
  • Les réels (real)
  • Les caractères (char)
  • Les chaines de caractères (string)
  • Les booléens (boolean)

En informatique, une variable possède à un moment donné une valeur et une seule.

On déclare une variable en PASCAL avec "VAR" avant d'écrire BEGIN du programme principal

    5) Les structures algorithmiques

    Cela concerne la conceténation et les imbrications de séquences et ruptures de séquences. Elle forme ainsi une partie d'un programme.

    Catégories

      Ruptures de séquences :
  • Rupture conditionnelle et non répétitive : IF / CASE OF
  • Rupture répétitive : FOR / WHILE / REPEAT
    • Ruptures conditionnelles :

    IF ... THEN ... ENDIF

    IF ... THEN ... ELSE ... ENDIF

    CASE (var) OF ...

    CASE (var) OF ... ELSE

    Ruptures répétitives :

    FOR (var) TO (var) DO ... ENDFOR

    WHILE (var) DO ... UNTIL

    REPEAT ... UNTIL (var)

    6) Les procédures et les fonctions

    Dans certains cas le programmeur peut vouloir décrire ou/et sauvegarder un enchainement d'actions qui n'existe pas de façon standard dans le compilateur. Cet usage est pratique lors de la réalisation d'un programme complexe. On décompose donc le programme en mini-blocs ayant chacun un rôle bien précis. Ce sont des PROCEDURES et des FONCTIONS.

    Les procédures

    C'est le déroulement classique d'un processus. On déclare une procédure dans la partie réservée aux variables.

    Déclaration d'une procédure :

    PROCEDURE [(liste des paramètres : type)]
    VAR : liste des variables
    BEGIN {corps de la procédure}
    {LISTE DES ACTIONS}
    END

    Ensuite, il suffit d'écrire le nom de la procédure dans le bloc principal pour déclencher la liste des actions décrites. Les variables déclarées à l'intérieur de la procédure sont inutilisables à l'extérieur du bloc.

    Les fonctions

    De même, que les procédures, une fonction est un sous programme. Mais la fonction à pour but principal d'effectuer un calcul et de renvoyer un résultat. On déclare une procédure dans la partie reservée aux variables.

    Déclaration d'une fonction :

    FUNCTION [(liste des paramètres : type)] : type_fonction
    VAR : liste des variables
    BEGIN {corps de la fonction}
    {LISTE DES ACTIONS}
    nom_de_la_fonction:=resulat_des_calculs
    END

    Le type de la fonction est le type de la valeur qui est renvoyée par la fonction. Les variables déclarées à l'intérieur de la fonction sont inutilisables à l'extérieur du bloc, tout comme la procédure.

    7) Les paramètres

      Avant de parler des paramètres, il faut parler des différentes variables => les variables globales et les variables locales :
  • Variable globale : Elle est définie dans l'en-tête du programme principal et est utilisable dans n'importe quel sous programme. Mais s'il existe une variable dans un sous programme qui porte le même nom qu'une variable globale alors c'est la variable locale qui sera prise en compte.
  • Variable locale : Elle est définie à l'intérieur d'un sous programme et sa portée est limitée à ce sous programme.
      • Maintenant, pour en revenir aux paramètres, ceux-ci sont considérés comme des variables locales. Ce sont des valeurs du bloc principal dont le sous programme a besoin pour éxécuter les instructions du programme principal. Il y a 2 types de paramètres :
  • Les paramètres formels : Ce sont les valeurs que recevra le sous-programme pour pouvoir fonctionner.
  • Les paramètres effectifs : Ce sont des valeurs réelles et actuelles reçues par le sous-programme au cours de l'éxécution du programme principal.

    • Chaque paramètre doit être identifié et doit avoir un type. Les paramètres effectifs peuvent être des variables ou des constantes.Si c'est une variable :

    • 1) Utiliser la valeur de la variable sans prendre en compte les modifications subies dans le programme principal => On parle de passage de paramètre par valeur
    • 2) Utiliser la variable en lui attribuant les modifications subies par le sous programme et le programme principal

      • A) Passage de paramètre par valeur :

      PROCEDURE (param1: type1;param2:type2);
      FUNCTION (param1: type1;param2:type2):type;

      B) Passage de paramètre par adresse : (Le paramètre formel se substitue au paramètre effectif lors de l'éxécution du sous programme et la variable prend la nouvelle valeur)

      Pour cela on utilise le mot VAR avant la déclaration du paramètre. On peut utiliser passer des paramètres par valeur et par adresse dans le même sous programme.

      FUNCTION (VAR param1: type1;param2:type2):type;
